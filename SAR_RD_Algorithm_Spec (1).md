## 4. SAR回波仿真数学模型
定义场景中第 $i$ 个点目标的坐标为 $(x_i, y_i, 0)$，雷达在方位时间 $\eta$ 的坐标为 $(0, V\eta, H)$。雷达与点目标的瞬时斜距为：
$$R_i(\eta) = \sqrt{x_i^2 + (V\eta - y_i)^2 + H^2}$$

## 5. RD成像算法处理流图
Code Agent 需要严格按照以下四步完成Range-Doppler算法。

### 5.1 距离向压缩 (Range Compression)
* 对原始二维回波矩阵的每一行进行快速傅里叶变换（FFT），转换至距离频域。
* 构建距离向匹配滤波器：$H_r(f_r) = \exp\left(j\pi rac{f_r^2}{K_r}ight)$。
* 将数据与滤波器相乘后，进行距离向逆傅里叶变换（IFFT），完成距离向脉冲压缩。

### 5.2 方位向FFT (Azimuth FFT)
* 对距离压缩后的数据沿着列方向进行FFT，将数据域转换至 Range-Doppler 域（距离-方位频域）。

### 5.3 距离走动与弯曲校正 (RCMC)
* **关键注意**：由于0.5m的高分辨率，距离走动与弯曲跨越的距离单元较多，不可忽略。
* 距离弯曲量公式推导近似为：$\Delta R(f_\eta) = R_0 \left( rac{1}{\sqrt{1 - (rac{\lambda f_\eta}{2V})^2}} - 1 ight) pprox rac{\lambda^2 R_0}{8 V^2} f_\eta^2$。
* 在 RD 域内，需采用 **Sinc插值核** 根据 $\Delta R(f_\eta)$ 对方位频域的数据进行距离向的重采样对齐，完成校正。

### 5.4 方位向压缩 (Azimuth Compression)
* 在 RCMC 之后的数据矩阵上，生成随最近斜距 $R_0$ 空变的方位向匹配滤波器：
    $$H_a(f_\eta) = \exp\left(jrac{4\pi R_0}{\lambda} \sqrt{1 - \left(rac{\lambda f_\eta}{2V}ight)^2} ight) pprox \exp\left(j\pi rac{\lambda R_0}{2 V^2} f_\eta^2ight)$$

* 对方位频域进行方位向IFFT，得到最终的高分辨率复数二维SAR图像。取其幅值即可输出二维成像结果。

## 6. Code Agent 开发执行规范
1.  **矩阵化运算避免显式循环**：SAR回波仿真涉及大规模的二维矩阵运算。由于Python及其他解释型语言中循环较慢，务必使用 `NumPy` 的矩阵广播（Broadcasting）机制来计算斜距与回波，取代对时间 $t$ 和 $\eta$ 的双重嵌套 `for` 循环。
2.  **可视化验证**：
    * 绘制回波实部/幅角二维图象。
    * 绘制距离向压缩后的二维图像，验证点目标的距离向聚焦。
    * 绘制RD最终成像后的二维幅度图（使用等高线或热力图），证明四个场景边缘点和内部大写字母（LWB）均清晰聚焦。
